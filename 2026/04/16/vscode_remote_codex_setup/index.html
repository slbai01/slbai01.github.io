VSCode + Remote SSH + Codex 插件配置指南（含 403 / reconnecting 解决方案）

一、问题背景

在使用 VSCode + Remote SSH 进行远程开发时，Codex 插件常见问题包括：

  • 本地 VSCode 可正常登录 Codex
  • 远端环境：
    • 报错 403 Forbidden
    • 或可以跳过登录，但提问时一直 reconnecting

二、问题本质

Codex 插件实际运行在远端服务器，而非本地环境

因此会产生两个核心问题：

  1. 登录态仅存在于本地，不在远端
  2. 远端服务器无法访问 OpenAI（网络受限）

三、总体解决思路

1）同步登录态（auth.json）
2）打通网络链路（SSH 反向代理）
